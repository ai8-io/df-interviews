# Technical interview — candidate materials

This repository contains what you need for **Round 3** (system design + live coding).

## Before your session

1. Read **[Round 3A — system design (candidate brief)](round_3a_system_design/CANDIDATE_BRIEF.md)** so you know format, timing, and the kinds of scenarios we explore.
2. For **Round 3B**, use the app under `round_3b_live_coding/deepflow-interview-chatbot/` — follow that folder’s `README.md` for setup. You may receive a fresh clone link closer to the interview; setup steps are the same.

## Structure

| Path | Purpose |
|------|--------|
| `round_3a_system_design/CANDIDATE_BRIEF.md` | What to expect in the design round and the scenario text(s) |
| `round_3b_live_coding/deepflow-interview-chatbot/` | Live coding exercise — runnable app + setup |

Bring your normal development environment (Docker, Node, Python as described in the chatbot README) and whichever **AI coding tools** you already use day to day.

If anything in setup fails, tell your interviewers as soon as you can so you do not lose time on environment issues.
